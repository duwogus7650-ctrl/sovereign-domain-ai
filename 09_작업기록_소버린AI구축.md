# 작업기록 — 소버린 도메인 AI 구축 (2026-06-20)

00~08 노트가 "결정"이라면, 이 문서는 그 결정을 **실제로 구현·검증한 작업의 기록**이다.
산출 코드는 `sovereign_ai/`, 비공개 저장소 `github.com/duwogus7650-ctrl/sovereign-domain-ai`.

## 0. 방향 확정 (노트 → 구현)
- 도메인: 모터설계·제어·제어보드설계·AI고장진단·강화학습.
- 두 가지 핵심 결정(요청에 따른 선택):
  - **RAG 우선** — "논문을 실제로 안다 + 오프라인". 소형 from-scratch 모델은 사실을 환각하므로,
    정확한 사실은 RAG가 담당하게 함.
  - **합법 오픈액세스 전량** — 유료 Q1 본문은 받지 않고(저작권), arXiv 풀텍스트 + OpenAlex
    메타·초록(Q1 포함 전 논문) + 공개 데이터셋 레지스트리로 커버.

## 1. RAG 스택 구축 (`sovereign_ai/`)
- `acquire/` arxiv_harvest·openalex_harvest — 합법 수집(중복 스킵=idempotent)
- `ingest/build_index` — PDF/초록 → 청크 → 임베딩(MiniLM 다국어) → FAISS, **증분·재개 가능**
- `serve/rag` — FAISS 검색 + Ollama(qwen3) 생성 + 출처 인용 + 환각 억제
- `distill/` — 오픈웨이트 교사 시퀀스레벨 증류(생성+LoRA)
- `cli` — acquire / index / ask / chat / distill-gen / status

## 2. 검증에서 잡은 버그 (전부 실행으로 발견·수정)
1. arXiv 따옴표 구문검색 → 단어 AND 검색 (0건 버그)
2. OpenAlex 인용수 정렬 → 제목·초록 정밀필터 (주제무관 고전 유입)
3. FAISS 한글경로 저장 실패 → 바이트 직렬화 우회
4. PDF 깨진 유니코드 → 토크나이저 크래시 (정제 추가)
5. 인덱스·메타 desync → 원자적 저장 + 정합성 assert (검색 출처 오류 방지)
6. 대용량 인덱싱 연속 종료(exit 127) → **원인=샌드박스의 프로세스 강제종료**.
   해법: 재개 가능 인덱싱 + 자동재개 드라이버 → 어떤 중단에도 진행분부터 이어감.

## 3. 코퍼스 구축·심화
- 1차: arXiv 732편(PDF 726) + OpenAlex 4,661편 → **32,286청크**.
- 심화: 도메인 쿼리 88개로 세분화(Halbach·SynRM·FCS-MPC·PINN·SAC 등), 넓이 중심 →
  OpenAlex **8,254편**, 인덱스 **36,301청크**. 새 니치 주제 검색 커버리지 확인.

## 4. 엔드투엔드 검증 (qwen3:8b, 완전 오프라인)
- 5개 도메인 한국어 질문 → 영어 Q1 논문 교차검색 → 근거기반 한국어 종합 답변 + `[n]` 출처.
- 예: 베어링 도메인적응, SiC 게이트드라이버, RL 제어 과제, PMSM 센서리스 — 모두 정상.

## 5. 증류(B) 트랙 — 프로덕션 준비 (실행은 GPU 머신)
- 합법 방식만: **폐쇄형(Claude/GPT/Gemini) 금지**, 오픈웨이트 교사만(약관·순수성).
- `distill-gen` 재개 가능·깨진JSON 복구, `train_lora --merge`, `requirements-distill.txt`,
  `docs/04_증류_실행_런북.md`(교사→LoRA→GGUF→Ollama→RAG 전 절차).
- self-distillation으로 `--base`를 from-scratch 모델로 바꾸면 "진짜 소버린 스택"(노트 06).
