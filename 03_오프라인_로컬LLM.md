# 오프라인 로컬 LLM

## 증류 모델이 오프라인에서 도는가? → 완전히 된다
핵심: **학습(증류)과 추론(실제로 돌리는 것)은 완전히 분리된 단계다.** "증류로 만들었다"는 사실 자체는 오프라인 가능 여부와 무관하다.

- 학습(증류): GPU와 teacher(또는 teacher가 만든 데이터)가 필요한 무겁고 일회성 작업.
- 결과물: 모델 파일 하나 = 가중치 + 구조 정보. Ollama라면 GGUF 파일.
- 추론: 파일을 메모리에 올려 행렬 곱을 돌리는 순수 로컬 연산. 인터넷이 전혀 개입하지 않음.

그래서 비행기용 오프라인 에이전트가 성립한다 — 출발 전에 `ollama pull`로 모델(예: qwen2.5-coder)을 받아두면, 비행 내내 CPU로 추론하는 데 네트워크가 필요 없다. 유일한 온라인 단계는 그 pull뿐.

## 증류는 오프라인에 유리하다
큰 모델 지식을 작은 모델로 압축하는 게 증류의 목적이라, 노트북·CPU에서 돌아갈 크기가 나온다. 실제로 작은 모델 상당수가 증류의 산물:
- DeepSeek-R1의 추론 능력을 작은 Qwen·Llama에 증류한 R1-Distill 계열
- Gemini 계보의 지식을 받은 Gemma

어떤 방식으로 만들어졌든, 일단 받아두면 오프라인 추론은 동일하게 된다.
