# 지식 증류 (Knowledge Distillation)

## 핵심 발상
정답 라벨만으로 작은 모델을 학습시키면 샘플당 신호가 사실상 "정답 1비트"뿐이다. 반면 잘 학습된 큰 모델(teacher)은 전체 확률 분포("고양이 85%, 개 10%, 여우 4%…")를 내놓는데, 여기엔 클래스 간 유사성 정보(예: 손글씨 "2"가 "7"과는 닮고 "9"와는 덜 닮음)가 담겨 있다. Hinton은 이를 **dark knowledge**라 불렀고, 증류의 본질은 student가 이 **soft target**을 모방해 샘플당 훨씬 풍부한 그래디언트를 받게 하는 것이다.

## 온도(Temperature)
잘 학습된 teacher의 softmax는 정답 하나에 확률이 거의 쏠려(예: 0.99) 미묘한 구조가 묻힌다. 그래서 softmax에 온도 T를 넣는다: `p_i = softmax(z_i / T)`. T를 키우면 분포가 부드러워져 작은 로짓들의 구조가 드러난다. 학습 때는 teacher·student 둘 다 높은 T를, 추론 때는 T=1을 쓴다.

## 손실 함수
```
L = α · CE(student, 정답) + (1 − α) · T² · KL(student_soft ∥ teacher_soft)
```
- 앞 항(CE): 진짜 정답을 맞히는 일반 교차엔트로피(정답항)
- 뒤 항(KL): teacher의 부드러운 분포를 따라가는 증류항
- T²를 곱하는 이유: 온도 때문에 작아진 그래디언트 크기 보정
- 가장 중요한 점: **그래디언트는 student로만 흐른다. teacher는 끝까지 고정(frozen).** 학습이 끝나면 teacher는 버리고 가벼운 student만 가져간다.

## 무엇을 모방하느냐
- 출력(response/logit) 기반: teacher의 최종 분포만 맞춤(고전적)
- 특징(feature) 기반: 중간 레이어 은닉 표현·어텐션 맵까지 정렬(FitNets 등)
- 관계(relation) 기반: 레이어 간/샘플 간 관계 구조 전달

## LLM에서의 증류
- 화이트박스(logit) 증류: teacher 로짓에 직접 접근 가능할 때 토큰별 전체 분포를 맞춤. 같은 토크나이저의 오픈웨이트 teacher일 때 가능.
- 블랙박스(시퀀스 레벨) 증류: API 모델처럼 출력 텍스트만 볼 때. 사실상 teacher로 합성 데이터셋을 만들어 student를 SFT(예: Alpaca).
- exposure bias: teacher의 정답 시퀀스로만 학습하면 추론 때 자기 출력 위에서 취약. → on-policy 계열(MiniLLM의 reverse KLD, GKD)로 student가 자기 출력 위에서 학습하되 teacher가 채점.
