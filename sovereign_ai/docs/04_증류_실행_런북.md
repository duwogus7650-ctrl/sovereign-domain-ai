# 증류 실행 런북 (GPU 머신용)

오픈웨이트 교사 → 도메인 SFT 데이터 → 학생 LoRA → GGUF → Ollama → RAG가 그 학생 모델 사용.
**이 전체는 GPU 머신에서 돌린다**(이 개발 환경은 8B 장시간 프로세스를 강제종료함). 합법성·원칙은 `docs/03_증류_프런티어모방.md` 참고 — 폐쇄형(Claude/GPT/Gemini) 사용 금지.

---

## 0. 사전 준비 (GPU 머신)
```bash
# 코어 + 증류 의존성
pip install -r requirements.txt
pip install -r requirements-distill.txt    # transformers/peft/trl/accelerate/datasets

# Ollama + 교사 모델 (클수록 데이터 품질↑). GPU VRAM에 맞게 선택.
ollama pull qwen3:14b      # 또는 qwen3:32b / qwen2.5:32b 등
```
> 인덱스(chunks.jsonl)가 있어야 한다. 없으면 먼저 `acquire`→`index` (README 참고).

## 1. 도메인 SFT 데이터 생성 (재개 가능)
교사가 코퍼스 청크를 근거로 (질문, 답변) 쌍을 만든다.
```bash
python -m sovereign.cli distill-gen --teacher qwen3:14b --n 1000
```
- `data/sft/domain_sft.jsonl` 에 한 줄당 한 쌍씩 적재. **중간에 멈춰도 재실행하면 이어서** 생성.
- 품질 점검: 몇 줄 열어 질문이 구체적이고 답변이 근거에 충실한지 확인. 거친 쌍이 많으면 교사를 더 큰 모델로.
- 권장 규모: 도메인 특화는 500쌍 이상이 분기점, 1,000~3,000쌍이면 충분히 말투·추론 전이.

## 2. 학생 LoRA 학습 (+병합)
```bash
python sovereign/distill/train_lora.py \
  --base Qwen/Qwen2.5-0.5B-Instruct \
  --data data/sft/domain_sft.jsonl \
  --out data/student_lora \
  --epochs 3 --batch 2 --grad_accum 8 --merge
```
- `--merge`: 학습 후 LoRA를 base에 병합해 `data/student_lora/merged/` 에 풀 모델 저장(GGUF 변환용).
- **self-distillation(진짜 소버린 스택)**: `--base`를 *당신의 from-scratch 모델 경로*로 바꾸면 됨(노트 06).
- VRAM 부족 시 `--batch 1 --grad_accum 16`, 또는 4bit는 `bitsandbytes` 추가 후 코드에 `load_in_4bit` 적용.

## 3. GGUF 변환 (llama.cpp)
```bash
git clone https://github.com/ggerganov/llama.cpp && cd llama.cpp
pip install -r requirements.txt
python convert_hf_to_gguf.py ../sovereign_ai/data/student_lora/merged \
  --outfile sovereign-student.gguf --outtype q8_0
# 더 작게: --outtype q4_k_m (양자화)
```

## 4. Ollama 등록
`Modelfile` 작성:
```
FROM ./sovereign-student.gguf
PARAMETER temperature 0.2
PARAMETER num_ctx 8192
SYSTEM "너는 모터·제어·전력전자·고장진단·강화학습 도메인 전문가다. 근거에 충실히 한국어로 답한다."
```
```bash
ollama create sovereign-student -f Modelfile
ollama run sovereign-student "PMSM 센서리스 제어 저속 영역 문제는?"   # 단독 점검
```

## 5. RAG가 학생 모델을 쓰게 연결
`config.yaml`:
```yaml
llm:
  model: sovereign-student
```
```bash
python -m sovereign.cli chat
```
이제 **학생(도메인 말투·추론) + RAG(정확한 근거·출처)** 하이브리드로 동작한다.

---

## 평가(권장)
학습 전/후를 같은 도메인 질문 셋으로 비교 — 답변이 더 도메인스럽고 간결해졌는지, 출처 인용은 유지되는지.
사실 정확도는 RAG가 담당하므로 학생은 환각이 늘지 않는 선에서 '말투·추론'만 좋아지면 성공.

## 로드맵상 위치
- 지금: 오픈웨이트 교사 → 실용 학생 (**파생**, 합법·실용).
- 다음: from-scratch 본모델(노트 06) → 그걸 교사로 self-distillation → **진짜 소버린 스택**.
