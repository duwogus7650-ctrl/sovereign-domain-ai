# 실행 가이드

## 0. 사전 준비
- Python 3.10+
- `pip install -r requirements.txt` (faiss-cpu, sentence-transformers 포함 — 첫 설치 시 torch 동반, 수백 MB)
- 로컬 LLM: [Ollama](https://ollama.com) 설치 후 `ollama pull qwen3:8b`
- (증류·학습 트랙은) GPU 권장 + requirements.txt의 주석 처리된 의존성 해제

## 1. 수집 (온라인 — 단 한 번)
```bash
python -m sovereign.cli acquire-arxiv      # 도메인별 arXiv 풀텍스트+메타
python -m sovereign.cli acquire-openalex   # Q1 포함 전 논문 서지+초록
python -m sovereign.cli datasets           # 공개 데이터셋 링크 출력(수동 다운로드)
```
- 처음엔 `config.yaml`의 `acquire.arxiv_max_per_query`(기본 40)를 작게 유지해 파이프라인을 검증.
- 재실행해도 중복은 자동 스킵(idempotent). 한도를 늘려 다시 돌리면 증분 수집.
- API 예의: `request_delay_sec`(기본 3초) 준수. 너무 공격적으로 당기지 말 것.

## 2. 인덱싱 (임베딩 모델 1회 다운로드 후 오프라인)
```bash
python -m sovereign.cli index                 # 한 번에
python -m sovereign.cli index --budget 600     # 600초만 처리하고 저장(이어서 재실행 가능)
```
- `data/pdf` + `data/my_papers` + `data/meta/*.jsonl`(초록)을 합쳐 청크→임베딩→FAISS.
- 산출: `data/index/faiss.index`, `data/index/chunks.jsonl`, `data/index/info.json`.
- **재개 가능(resumable)**: 중간에 멈추거나 죽어도 다시 `index`를 실행하면 처리완료 문서는
  건너뛰고 이어서 진행한다. faiss와 chunks.jsonl은 항상 함께 저장돼 정합성이 보장된다(자기검증 assert).
- **임베딩 모델**: CPU 기본값은 MiniLM-L12 다국어(384차원, CPU 16코어 ~16청크/초).
  GPU가 있으면 `config.yaml`에서 `BAAI/bge-m3`(1024차원, 더 정확)로 바꾸고 재인덱싱 권장.
- **참고(현재 구축분)**: arXiv 풀텍스트 726편 + 초록 5,489편 → **32,286청크** 색인 완료.
- **Windows 주의**: torch 임베딩처럼 무거운 프로세스는 환경에 따라 강제종료될 수 있다.
  그럴 땐 위 `--budget` 으로 나눠 돌리거나, 죽으면 다시 `index`를 실행(재개됨)하면 된다.

## 3. 질의 (완전 오프라인)
```bash
python -m sovereign.cli ask "코깅 토크를 줄이는 슬롯/극 조합 설계 원리는?"
python -m sovereign.cli chat
python -m sovereign.cli status
```
- 답변은 검색된 근거 청크에 기반하며 `[n]` 출처를 표시. 근거 없으면 "자료에 없음"이라고 답함(환각 억제).

## 4. (선택) 증류로 도메인 말투/추론 전이
```bash
python -m sovereign.cli distill-gen        # 오픈웨이트 교사가 SFT 데이터 생성
python sovereign/distill/train_lora.py --base Qwen/Qwen2.5-0.5B-Instruct
```
- 학습된 학생을 GGUF로 변환해 `ollama create`로 등록하고 `config.yaml > llm.model`을 그 모델로 바꾸면,
  RAG 엔진이 그대로 당신의 도메인 학생 모델을 호출한다.

## 트러블슈팅
| 증상 | 원인/해결 |
|---|---|
| `인덱스가 없습니다` | 먼저 `index` 실행. 그 전에 `acquire` 또는 `data/my_papers`에 PDF 필요 |
| `LLM 호출 실패` | Ollama 실행 여부 / `ollama pull qwen3:8b` 여부 확인 |
| 임베딩이 느림 | `config.yaml > embedding.device: cuda`(GPU) 또는 더 작은 모델 |
| arXiv 0건 | 네트워크/쿼리 확인. 쿼리 따옴표·카테고리 조합 점검 |
| 청크 0개 | PDF 추출 실패(스캔본일 수 있음) — 초록 기반으로라도 색인됨 |

## 단계별 로드맵 (노트 06번과 정렬)
- **Stage 0 (지금)**: RAG 파이프라인 가동 — 적은 코퍼스로 전 과정 손에 익히기.
- **Stage 1**: 코퍼스 확장(한도↑, 데이터셋 추가) + 오픈웨이트 교사 증류로 실용 학생.
- **Stage 2**: from-scratch 본모델(노트 06) → self-distillation → "진짜 소버린 스택".
