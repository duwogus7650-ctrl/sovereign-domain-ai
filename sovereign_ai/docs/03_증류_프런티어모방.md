# 증류로 "프런티어 모델 모방" — 합법적으로 하는 법

> 요청: *"증류 방식으로 프런티어 모델을 모방하기도 해봐."*
> 이 문서는 그걸 **당신의 노트가 이미 내린 결론을 깨지 않으면서** 실제로 하는 방법이다.

## 먼저, 두 갈래를 분명히 (노트 05번의 그 지점)

| 증류 대상 | 결과 | 합법성 | 채택? |
|---|---|---|---|
| **폐쇄형 프런티어**(Claude·GPT·Gemini 출력으로 학습) | "이름뿐인 소버린" — 종속·천장 | **약관 위반** ("출력으로 경쟁모델 학습 금지") | ❌ 안 함 |
| **오픈웨이트 교사**(Qwen3·Llama·DeepSeek, 라이선스 허용) | 파생이지만 합법·실용 | ✅ 라이선스 준수 시 | ✅ 채택 |
| **자체 from-scratch 모델 self-distill** | 진짜 소버린 스택 (경량화·배포 도구) | ✅ | ✅ 최종 목표 |

**왜 폐쇄형(나, Claude 포함)을 안 쓰나:** ① Anthropic·OpenAI 약관이 "출력으로 경쟁 모델 학습"을 금지 — 어기면 그 위에 쌓은 소버린이 처음부터 위법. ② 당신 노트의 결론 그대로 — 증류 학생은 교사를 못 넘고("모방 후 추월 불가"), 한국어 약점까지 물려받아 *이름뿐인 소버린*이 된다. **그래서 "프런티어 모방"은 폐쇄형 복제가 아니라, 합법 교사로 "프런티어급 행동/추론 방식"을 모방하는 것으로 구현한다.**

## 채택한 방식: 시퀀스레벨(블랙박스) 증류 = 교사가 학습데이터를 만든다

화이트박스 logit 증류는 교사 가중치가 필요하다(오픈웨이트만 가능, 무겁다).
대신 **교사가 도메인 질문에 모범답안을 생성 → 그 (질문, 답) 쌍으로 학생을 SFT** 하는
시퀀스레벨 증류를 쓴다. Alpaca·distill 계열이 쓰는 바로 그 방법이고, 우리 코퍼스와 자연스럽게 맞물린다.

```
RAG 코퍼스 청크  ──►  오픈웨이트 교사(Ollama, 예: qwen3:14b)
   (모터/제어/RL)        │  "이 근거로 도메인 Q&A를 만들어라"
                         ▼
              data/sft/domain_sft.jsonl   (질문·근거기반 답변 쌍)
                         │  LoRA SFT
                         ▼
              학생 모델 (소형 / 당신의 from-scratch 모델)
                         │  배포
                         ▼
   추론 시: 학생(도메인 말투·추론) + RAG(정확한 사실)  ← 하이브리드
```

핵심: **학생이 "사실"을 외우게 하지 않는다**(소형 모델은 환각). 학생은 *도메인 추론·설명 방식*을 배우고,
정확한 수식·수치는 추론 시 RAG가 근거로 댄다. 이게 노트의 순수성 지향과 "논문을 안다"를 동시에 만족시키는 지점.

## 파이프라인 (이 레포 구현)

1. `python -m sovereign.cli distill-gen` — 교사가 코퍼스 기반 SFT 데이터 생성
   - 교사 모델은 `config.yaml > distill.teacher_model` (오픈웨이트, Ollama)
   - 근거 청크를 주고 (지시, 모범답안)을 JSONL로 적재
2. `python sovereign/distill/train_lora.py` — 학생에 LoRA SFT (GPU 권장)
   - 학생 base: 오픈웨이트 소형(Qwen3-0.6B 등) **또는** 당신의 from-scratch 모델
3. GGUF 변환 → Ollama 등록 → 오프라인 서빙(기존 RAG 엔진이 그대로 그 모델을 호출)

## 정직한 한계
- 오픈웨이트 교사로 만든 학생은 **여전히 '파생'**이다(노트 분류상). 100% 순수 from-scratch가 목표라면,
  교사를 *당신의* from-scratch 모델로 바꿔 self-distillation 하는 순간에만 "진짜 소버린 스택"이 된다.
- 따라서 권장 로드맵: **(지금) 오픈웨이트 교사로 실용 학생 → (나중) from-scratch 본모델 → self-distill 경량화.**
  실용성으로 출발해 순수성으로 수렴하는 길. 둘을 굳이 양자택일하지 않는다.
